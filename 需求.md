# 任务目标
请在当前空白工作区中，从零开始搭建一个完整的、独立的 Vue 3 Web 项目。该项目的核心功能是一个类似于 DBeaver 的轻量级 DuckDB 数据库可视化管理平台。

# 核心技术栈规范 (必须严格遵守)
1. 框架：Vue 3 + TypeScript + Vite
2. UI 框架：Naive UI (`naive-ui`)
3. 图标库：`@vicons/ionicons5`
4. 代码编辑器：`vue-codemirror` (配合 `@codemirror/lang-sql`)
5. 网络请求：`axios`
6. 样式：使用 `less`，推荐使用内联 `<style scoped lang="less">`。

# 后端接口规约（API Contract）
目标服务器地址暂定为：`http://10.86.151.94:8689`。
**非常重要：所有接口的 `querySql` 参数必须通过 URL Query (Params) 传递，即便是 POST 请求，也不要把 SQL 放在 Body 里！**
1. 查询 (GET) `/bdmp-connector-master/duckdb/query?querySql={sql}`：返回纯 JSON 数组（动态列）。
2. 更新 (POST) `/bdmp-connector-master/duckdb/update?querySql={sql}`：返回 `{"code", "message", "data"}`。
3. 结构变更 (POST) `/bdmp-connector-master/duckdb/execute?querySql={sql}`：返回 `{"code", "message", "data"}`。

# 项目执行步骤引导（请严格按顺序执行，并请求创建对应文件）

## 第一步：项目初始化与依赖配置
请直接创建基础配置文件，无需使用 npm init 交互式命令，直接生成以下文件：
1. `package.json`：配置好上述技术栈的 dependencies 和 devDependencies，以及 dev/build 脚本。
2. `tsconfig.json` 和 `tsconfig.node.json`：标准的 Vue 3 TS 配置。
3. `index.html`：项目入口 HTML。
4. `vite.config.ts`：**【关键】** 配置 server proxy，将 `/bdmp-connector-master` 代理到 `http://10.86.151.94:8689`，以解决本地开发的跨域问题。

## 第二步：核心应用骨架构建
创建以下基础目录和文件：
1. `src/main.ts`：挂载 Vue 应用，并全局引入 Naive UI 的样式（如果需要）。
2. `src/api/duckdb.ts`：配置 axios 实例，统一处理 baseURL（指向代理路径），并封装 `queryData`, `updateData`, `executeSql` 三个方法，注意使用 `encodeURIComponent` 处理 SQL 字符串。
3. `src/App.vue`：作为主容器，使用 `<n-message-provider>` 和 `<n-dialog-provider>` 包裹整个应用，方便全局消息提示。

## 第三步：核心业务组件开发 (src/views/DuckDbConsole.vue)
在此组件中实现经典的三栏式数据库管理界面，使用 `<n-layout has-sider>`：
1. **左侧栏 (数据库对象树)**：
   - 挂载时调用 GET `show tables` 渲染表名节点。
   - 点击表名时懒加载，调用 GET `PRAGMA table_info('表名')` 获取字段信息作为子节点。
2. **右上区 (SQL 编辑器)**：
   - 集成 CodeMirror 6，提供 SQL 语法高亮。
   - 提供三个操作按钮（查询、更新、结构变更），分别调用对应的 API。
3. **右下区 (结果与日志区)**：
   - 使用 `<n-tabs>`。
   - **数据 Tab**：根据查询返回的 JSON 数组，动态提取 keys 生成 `columns`，并使用 `<n-data-table>` 渲染动态表格。
   - **日志 Tab**：显示 API 调用的成功/失败响应信息（message）。
