## 二、核心应用骨架构建

### 2.1 [`src/main.ts`](src/main.ts)

Vue 应用入口文件，创建并挂载应用到 `#app`。

### 2.2 [`src/api/duckdb.ts`](src/api/duckdb.ts)

API 接口封装层，关键设计：

- **axios 实例**：`baseURL` 指向 `/bdmp-connector-master/duckdb`（走 Vite 代理）
- **三个接口方法**：
  | 方法 | HTTP 方法 | 端点 | 说明 |
  |------|-----------|------|------|
  | `queryData(sql)` | GET | `/query` | 查询数据，返回 `any[]` |
  | `updateData(sql)` | POST | `/update` | 更新数据，返回 `{code, message, data}` |
  | `executeSql(sql)` | POST | `/execute` | 结构变更，返回 `{code, message, data}` |
- **重要**：所有 `querySql` 参数通过 `encodeURIComponent` 处理后放入 URL Query (Params)，即使是 POST 请求也不放入 Body。

### 2.3 [`src/App.vue`](src/App.vue)

根组件，使用 `<n-message-provider>` 和 `<n-dialog-provider>` 包裹 `DuckDbConsole` 组件，提供全局消息提示和对话框能力。同时包含了全局样式重置。

## 三、核心业务组件开发

### [`src/views/DuckDbConsole.vue`](src/views/DuckDbConsole.vue)

实现经典的三栏式数据库管理界面，使用 `<n-layout has-sider>` 布局：

#### 3.1 左侧栏 - 数据库对象树

- **挂载时**：调用 `GET show tables` 渲染所有表名节点
- **点击表名时**：懒加载调用 `PRAGMA table_info('表名')` 获取字段信息（字段名 + 类型），作为子节点展示
- 包含刷新按钮，支持手动重新加载表列表
- 自定义图标：表节点使用 `TableChartOutline`（蓝色），字段节点使用 `CodeOutline`

#### 3.2 右上区 - SQL 编辑器

- 集成 CodeMirror 6 + `@codemirror/lang-sql`，提供 SQL 语法高亮
- 默认预填 `show tables` 语句
- 三个操作按钮：
  - **查询**（蓝色）- 调用 `queryData`
  - **更新**（橙色）- 调用 `updateData`
  - **结构变更**（红色）- 调用 `executeSql`
- 按钮带 loading 状态，执行中不可重复点击

#### 3.3 右下区 - 结果与日志区

使用 `<n-tabs>` 分为两个 Tab：

- **数据 Tab**：
  - 根据查询返回的 JSON 数组，动态提取第一行的 keys 生成表格 `columns`
  - 使用 `<n-data-table>` 渲染动态表格（bordered、ellipsis）
  - 未执行查询时显示空状态提示
- **日志 Tab**：
  - 深色终端风格日志容器
  - 按时间倒序记录 API 调用结果
  - 颜色区分：成功（绿色）、失败（红色）、信息（蓝色）
  - 每条日志包含时间戳和消息内容
